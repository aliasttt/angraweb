# راهنمای کامل نصب و راه‌اندازی

## پیش‌نیازها

- Python 3.8 یا بالاتر
- pip (مدیر بسته Python)

## مراحل نصب

### 1. نصب وابستگی‌ها

```bash
pip install -r requirements.txt
```

یا به صورت دستی:

```bash
pip install Django>=5.1.2 Pillow>=10.0.0
```

### 2. اجرای Migrations

```bash
python manage.py makemigrations
python manage.py migrate
```

### 3. ایجاد کاربر ادمین

```bash
python manage.py createsuperuser
```

نام کاربری، ایمیل و رمز عبور را وارد کنید.

### 4. ایجاد داده‌های اولیه (اختیاری)

```bash
python manage.py create_initial_data
```

این دستور خدمات و پکیج‌های اولیه را ایجاد می‌کند.

### 5. اجرای سرور

```bash
python manage.py runserver
```

سپس به آدرس `http://127.0.0.1:8000` بروید.

## دسترسی به پنل ادمین

```
http://127.0.0.1:8000/admin/
```

با نام کاربری و رمز عبوری که در مرحله 3 ایجاد کردید وارد شوید.

├── static/                   # فایل‌های استاتیک
│   ├── css/                  # فایل‌های CSS
│   ├── js/                   # فایل‌های JavaScript
│   └── ...
├── media/                    # فایل‌های آپلود شده
│   ├── services/             # تصاویر خدمات

## استفاده از پنل ادمین

### اضافه کردن خدمات جدید

1. وارد پنل ادمین شوید (`/admin/`)
2. روی "Services" کلیک کنید
3. "Add Service" را بزنید
4. اطلاعات را پر کنید:
   - عنوان (به زبان‌های مختلف)
   - نوع خدمت
   - توضیحات
   - آیکون (مثلاً: `fas fa-code`)
   - تصویر (اختیاری)
   - ترتیب نمایش
5. "Save" را بزنید

### اضافه کردن پکیج جدید

1. "Packages" را باز کنید
2. "Add Package" را بزنید
3. اطلاعات پکیج را وارد کنید
4. ویژگی‌های پکیج را در بخش پایین اضافه کنید
5. ذخیره کنید

### اضافه کردن پروژه جدید

1. "Projects" را باز کنید
2. "Add Project" را بزنید
3. اطلاعات پروژه را وارد کنید:
   - عنوان
   - نوع پروژه
   - توضیحات
   - تصویر
   - لینک پروژه (اگر دارد)
   - دامنه
   - تکنولوژی‌ها (جدا شده با کاما)
4. ذخیره کنید

### مدیریت پیام‌های تماس

1. "Contact Messages" را باز کنید
2. پیام‌های جدید را مشاهده کنید
3. وضعیت پیام‌ها را تغییر دهید:
   - جدید
   - خوانده شده
   - پاسخ داده شده
   - آرشیو شده

### مدیریت درخواست‌های قیمت

1. "Quote Requests" را باز کنید
2. درخواست‌های جدید را بررسی کنید
3. وضعیت را تغییر دهید و یادداشت اضافه کنید

## صفحات موجود

- `/` - صفحه اصلی
- `/services/` - لیست همه خدمات
- `/service/<slug>/` - جزئیات یک خدمت
- `/web-design/` - صفحه طراحی وبسایت
- `/mobile-app/` - صفحه طراحی اپلیکیشن موبایل
- `/ecommerce/` - صفحه فروشگاه اینترنتی
- `/seo/` - صفحه سئو و بهینه‌سازی
- `/hosting/` - صفحه هاستینگ و دامنه
- `/ui-ux/` - صفحه طراحی UI/UX
- `/packages/` - لیست پکیج‌ها
- `/projects/` - لیست پروژه‌ها
- `/project/<slug>/` - جزئیات پروژه
- `/about/` - درباره ما
- `/resume/` - رزومه و گواهینامه‌ها
- `/contact/` - تماس با ما
- `/quote/` - درخواست قیمت
- `/blog/` - وبلاگ
- `/blog/<slug>/` - جزئیات پست وبلاگ
- `/login/` - ورود
- `/register/` - ثبت‌نام
- `/logout/` - خروج
- `/lang/<lang_code>/` - تغییر زبان (tr, en, fa, ar)

## نکات مهم

1. **فایل‌های استاتیک**: فایل‌های CSS و JS در پوشه `static/` هستند
2. **فایل‌های رسانه**: تصاویر و فایل‌های آپلود شده در `media/` ذخیره می‌شوند
3. **دیتابیس**: در حالت توسعه از SQLite استفاده می‌شود
4. **زبان‌ها**: سایت از 4 زبان پشتیبانی می‌کند (ترکی، انگلیسی، فارسی، عربی)

## تولید (Production)

برای استقرار در سرور واقعی:

1. تنظیمات `settings.py` را تغییر دهید:
   - `DEBUG = False`
   - `ALLOWED_HOSTS` را تنظیم کنید
   - `SECRET_KEY` را تغییر دهید

2. فایل‌های استاتیک را جمع‌آوری کنید:
   ```bash
   python manage.py collectstatic
   ```

3. از دیتابیس PostgreSQL یا MySQL استفاده کنید

4. از سرور وب مانند Nginx و Gunicorn استفاده کنید
